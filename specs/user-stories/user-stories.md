# User Stories

## Core Gameplay

- As a user, I'm able to visualize the code in a small side-by-side container next to the code that I'm writing because it will help me build a better understanding for it.
- As a website user, I want a game that gives clear prompts and visible tracking progress because it allows me to grow my understanding.
- As a user, I want to have different types of websites (prompts) to practice on so I can gain hands-on experience writing code for all types of websites.
- As a user, I want to be able to choose the level of difficulty (perhaps 3 stages) so I can better choose which one fits me.
- As a user, I want a final generated webpage/screenshot at the end of the game so that I can review what I created.
- As a user, I want a separate countdown mode so that the game feels challenging and competitive.

## Progress & Metrics

- As a user, I want to have a list of metrics showing my error rate, syntax I learned, WPM, time, and other related metrics to see my progress and mistakes.
- As an analytical user, I want to know what keys I am messing up more so that I know to improve on these specific keys.

## UI & Accessibility

- As a user, I would like a light and dark theme because I might prefer a light or dark mode and having the website look nice encourages me to use the site.
- As a mobile user, I want to be able to type small amounts of code (tags, keywords, etc) because typing symbols and long text sucks on mobile.
- As a mobile user, I want my game to feature vertical or landscape modes so it is more flexible for me to see.

## Developer Experience

- As a TA developer, I want to be able to clearly identify separate features and how I would go about adding new features because this is part of the grading criteria.
- As a TA developer, I want a README that clearly documents how to deploy the project because this is part of the grading criteria.
- As a developer, I want a main documentation that specifies the coding practices/naming conventions used in the repo so I can follow the coding standards of the organization.
- As a developer, I want to have a flowchart to visualize the overall architecture of the application so I can better understand where/how different features are made/being used in the repo.
- As a developer, I want a unit testing system so that as I implement features I can write tests alongside to ensure they are functional.
- As a developer, I want a local development environment because it allows me to test my code before pushing.
- As a developer, I want the team to have a development environment because it allows me to have my code tested by others before merging into main.

---

## Low Priority

### Personalization & Accessibility

- As a user who isn't a native English speaker, I want to be able to choose a different language for the website text (localization) so I can navigate the web page better.
- As a user with visual impairments, I want to adjust font size and contrast settings so that I can play the game comfortably without straining to read the text on screen.
- As an aspiring developer, I want to be able to learn various coding languages so that I can focus on what is most relevant to my learning goals.

### Engagement & Retention

- As a motivated user, I want to see my daily login streak so that I can feel rewarded for learning every day.
- As a forgetful user, I want to set daily reminders at a time I want so that I can remember to practice when I need to and stay consistent.
- As a user, I want background music and sound effects so that gameplay feels more immersive.
- As a user, I want animations when making the website so the website building is more satisfying.

### Data & Offline Support

- As a commuting student, I want the website to load and then not require internet so I can use the site even with an unstable connection.
- As a user, I want my data to be stored in a cache so I can use the website and see how my progress changes over time.
- As a user, I want to be able to save my overall account progress stored in 3rd party cloud storage so I am able to log in later on or on different devices.
