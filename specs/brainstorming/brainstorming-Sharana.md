# Sharana Sabesan - Brainstorming Document

## Concept
Chase down bugs, on the clock. Prevent opposing team shots by catching the "bug", and speed up your three-pointers using the same strategy. 

## Target Audience
This is for users who find it difficult learning to catch bugs, and find it hard to concentrate on doing so, have an action-powered game to do it! 

## Core Loop
The anxiety from the large countdown to keeps the user's senses heightened. They hear or see the buzzer went off, scramble to catch the bug before it is too late. If it's too late, the opposing team gets a point, if they find it in time, they get a point. 

## The 3 Key Features
- Stamina system -> LeBron has a power bar. If his energy is depleting, you are given more bugs, albeit easier (to score more of your stamina back, but it is also meant to overwhelm the user.) If his energy is high up you are given a harder bug, albeit only one. 
- Ranging difficulty -> Difficulty of bugs goes up as the user completes more "games" (aka levels). The user moves from reference errors, to something more difficult. They can also score points with some freebie easy ones to keep them engaged. 
- The "Game" system -> Each game is a level with ranging difficulty. Each game won, means a level has been completed, and the user moves up not only in difficulty, but also in powerups. 

## Unique Selling Point
Learning to understand and catch bugs is something CS students generally struggle with, plug into ChatGPT, and fail to actually understand how bug-finding and bug-fixing works. To make fixing bugs fun again, before AI tools, this game is the right tool. The USP is the unique market that is being targetted. 

## Inspiration / References
This game draws from real-life Basketball and from the legendary basketball champion: LeBron James. When he wins, all of us win. 

## Scope
**MVP:** What's the absolute minimum version that's playable/usable?
An MVP would look like a text-based chatbot game where the user is prompted with different questions and outputs to show the game is being played. We would have 3 rounds per game. Each round, a scoreboard and timer appear, as well as a chunk of code representing a bug you must catch.
